Jalan Jalan is a trip planning mobile application built for SDG 11: Sustainable Cities and Communities. The application helps users navigate and plan trips around Kuala Lumpur, Melaka, and Batu Pahat through GPS tracking, live weather information, local trip history, and community trip sharing. Developed using Jetpack Compose, the app integrates Room Database, Firebase Firestore, Retrofit, and GPS sensor technologies to support smarter and more sustainable urban mobility.

Projects

Applying concepts from labs into a real-world Android application.

```

Project 1 – JALAN JALAN

Problem Statement:
Modern mobile apps need persistent storage because ViewModel data disappears when the application is destroyed. This project integrates Room Database into the Jalan Jalan trip planner to provide reliable long-term storage.

Through this project, I learned how Entity, DAO, Database, Repository, and ViewModel components work together in the MVVM + Repository + Room architecture. I also understood the value of Flow and StateFlow for automatic UI updates and learned how Singleton design patterns ensure efficient and thread-safe database management.

Project 2 – JALAN JALAN

Problem Statement:
Many Malaysians struggle to plan trips efficiently without live weather information, location tracking, and community travel insights. Jalan Jalan solves this challenge by combining GPS, live weather APIs, and cloud-based trip sharing into a unified platform.

Building Project 2 taught me how to integrate local databases, cloud storage, GPS sensors, and live APIs into a single application. I also learned the importance of handling asynchronous operations carefully and implementing fallback mechanisms to ensure application stability in real-world situations.

Reflection & Learning Journey

```

From Lab 1 to Project 2, my understanding of Android development evolved from building simple static interfaces to designing a fully connected mobile application using local storage, cloud databases, APIs, and hardware sensors. Early concepts such as Coroutines, Flow, StateFlow, and Firebase initially felt complex, but organizing features using Repository and ViewModel layers made the architecture easier to understand and maintain.

The most challenging aspect was debugging asynchronous operations involving GPS services and network requests. Understanding why fallback values behaved differently between emulators and real devices taught me the importance of thorough testing and robust error handling.

This experience strengthened my appreciation for designing applications that handle real-world failures gracefully rather than only supporting ideal scenarios. Most importantly, it demonstrated how software development can contribute meaningfully to SDG 11 by supporting smarter and more sustainable urban mobility.
