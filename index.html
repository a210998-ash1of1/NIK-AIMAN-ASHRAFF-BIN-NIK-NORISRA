Home

Introduction: Jalan Jalan is a trip planning mobile application built for SDG 11: Sustainable Cities and Communities, designed to help users navigate and plan trips around Kuala Lumpur, Melaka, and Batu Pahat. The app combines real-time GPS location, live weather data, local trip history, and a community trip-sharing feature, allowing users to plan smarter, more informed journeys. Built with Jetpack Compose, it integrates four core technologies — Room database for local storage, Firebase Firestore for cloud sharing, Retrofit for live weather data, and GPS sensor hardware for location detection.

SDG: SDG 11 + sustainable cities and communities, because building smarter mobility and resilient infrastructure helps reduce congestion, pollution, and improve urban living quality.

Projects

Project 1 (Lab 4 + Working Features)

App Name: JALAN JALAN

Problem Statement: Modern mobile apps need to store data permanently so that information is not lost when the app is closed, but using only a ViewModel is not enough because ViewModel data is stored in memory and disappears once the app is destroyed. This project addresses that gap by integrating a Room database into the Jalan-Jalan trip planner app to enable persistent data storage.

Through this project, I learned how the Entity, DAO, Database, Repository, and ViewModel components work together in the MVVM + Repository + Room architecture to manage data flow cleanly. I also learned the importance of using Flow and StateFlow so the UI updates automatically whenever the underlying database changes, without needing manual refresh logic. Lastly, I understood the Singleton pattern's role in ensuring only one database instance exists, which prevents conflicts and keeps the app's data layer efficient and thread-safe.

Project 2 (Lab 5 + API + Sensors + Cloud)

App Name: JALAN JALAN

Problem Statement: Many Malaysians struggle with planning urban trips efficiently — without easy access to live weather conditions, real-time location tracking, and a way to learn from other travelers' routes, trip planning becomes inconvenient and uninformed. Jalan Jalan addresses this by integrating GPS, live weather data, and community-shared trips into a single, connected platform.

Building Project 2 taught me how to connect multiple data sources — local, cloud, and live internet APIs — into a single cohesive app using ViewModel and StateFlow. I learned that handling asynchronous operations like GPS and network calls requires careful state management (Loading/Success/Error) to avoid crashes and give users clear feedback. I also learned the importance of fallback handling — for example, defaulting to a safe location when GPS fails — to keep the app stable under real-world conditions.
